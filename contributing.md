# Contributing to FinnaCloud

Thank you for your interest in contributing to **FinnaCloud**!  
We welcome contributions of all kinds, code, documentation, bug reports, feature requests, and ideas. This guide will help you get started and ensure your contributions are effective and appreciated.

---

## **How You Can Contribute**

You can contribute in several ways:

- **Code**: Implement new features, fix bugs, or improve existing functionality.  
- **Documentation**: Improve README files, guides, or inline docs.  
- **Issues**: Report bugs, suggest features, or provide feedback.  
- **Testing**: Write or run tests to verify changes.  
- **Ideas & Design**: Share suggestions on architecture, usability, or ecosystem improvements.

---

## **Getting Started**

1. **Fork the repository** to your GitHub account.  
2. **Create a new branch** for your changes:  

   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes** in your branch.  
4. **Commit your changes** with clear messages:

   ```bash
   git commit -m "Add feature XYZ"
   ```

5. **Push your branch** to your fork:

   ```bash
   git push origin feature/your-feature-name
   ```

6. **Open a Pull Request** against the main branch. Include:  
   - Description of your changes  
   - Any related issues  
   - Steps to test or verify your changes  

---

## **Pull Request Guidelines**

- Keep commits **focused and logical**.  
- Write clear and concise **commit messages**.  
- Include **tests** when applicable.  
- Follow **project coding standards**.  
- Be responsive to feedback from maintainers.

---

## **Reporting Issues**

If you find a bug or issue:

1. Check existing issues to avoid duplicates.  
2. Open a new issue with:
   - Clear title  
   - Steps to reproduce  
   - Expected vs actual behavior  
   - Environment or version details if relevant  

---

## **Branching & Workflow**

We follow a **GitHub Flow** workflow:

- `main` always stable, production-ready  
- Feature branches for all work before PR  
- PRs must be reviewed before merging
